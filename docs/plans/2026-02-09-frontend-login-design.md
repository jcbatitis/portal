# Frontend Login Page Design

## Context

The portal has no frontend. We need a React app with a login page that talks to the existing `POST /login` backend endpoint.

## Tech Stack

- Vite + React + TypeScript
- Tailwind CSS v4
- shadcn components (`button`, `input`, `label`, `card`)

## Setup

Scaffold a Vite + React + TypeScript project in `frontend/`. Initialize shadcn (installs Tailwind and CSS config). Add four shadcn components: `button`, `input`, `label`, `card`.

Root `package.json` scripts:
- `dev:frontend` — `npm run dev -w frontend`
- `build:frontend` — `npm run build -w frontend`

Vite dev server proxies `/api` to `http://localhost:3000` so the login form can call the backend without CORS issues.

## Login Page

### Layout

Full-viewport flex container, centers child vertically and horizontally.

### Component: `LoginPage` (`components/login-page.tsx`)

Uses shadcn `Card` as wrapper with `max-w-[400px] w-full`:

- **CardHeader**: Title "Login" (no subtitle)
- **CardContent**: Two fields — `Label` + `Input` for username, `Label` + `Input` (type password) for password
- **CardFooter**: `Button` (full width) with text "Log in"

### Behavior

- On submit: POST to `/api/login` with `{ username, password }`
- On 200: redirect to `/` (no dashboard yet — just refreshes)
- On 401: show red error text below button — "Invalid credentials"
- On network/other errors: "Something went wrong"
- While loading: button text "Logging in...", button disabled
- State: `useState` for username, password, error, loading (no form library)
